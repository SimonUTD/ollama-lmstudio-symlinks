# 项目 Agent 指南（AGENTS.md）

## 语言

- 用户可见回复默认使用中文（除非用户明确要求其他语言）。

## Debug-First（禁止静默降级）

- 不要为了“跑起来”而添加隐藏的 fallback/边界规则/假成功路径。
- 失败必须显式暴露：返回错误、日志、测试失败都可以；不要吞错。
- 如果确实需要 guardrail（安全/隐私/用户明确要求），必须：
  - 显式可见、文档化、易于关闭，并提前征得用户同意。

## 工程质量基线

- 遵循 SOLID/DRY/YAGNI，命名清晰，抽象务实。
- 优先不可变数据：不要修改入参或全局状态；返回新值。
- 业务逻辑依赖注入：核心逻辑不直接 hardcode 外部实现（例如命令执行、文件系统、时间、网络）。

## 代码指标硬限制

- 单函数 ≤ 50 行（不含空行），超过必须拆分 helper。
- 单文件 ≤ 500 行，超过必须按职责拆分。
- 最大嵌套深度 ≤ 3（用早返回/guard clause 拉平）。
- 位置参数 ≤ 3，更多用 options/config struct。
- 圈复杂度 ≤ 10，超过必须拆分。
- 禁止魔法数字：提取为具名常量。

## 安全基线

- 禁止在源码写入任何密钥/API Key/凭证；用环境变量或系统配置。
- 所有外部输入都在边界处校验（用户输入、API 返回、文件内容）。

## 验证与测试

- 功能变更必须先写失败测试（TDD），再写实现。
- 声称“完成/通过”前必须运行验证命令并基于输出给出结论。
- 运行后台单测时设置硬超时 60 秒，避免卡死。

## 本项目约定（与需求相关）

- 同步的核心是 symlink：默认不覆盖普通文件；覆盖/修复 symlink 必须是显式开关行为。
- LM Studio → Ollama 仅支持 `.gguf`；非 `.gguf` 在 GUI 中置灰并说明原因。
- 可执行文件发现不要仅依赖 `PATH`：需支持固定路径探测与用户显式配置（macOS Finder 启动不加载 shell 环境变量）。
- GUI 的写接口（`/api/*` 的 POST/PUT）要求 `Content-Type: application/json`，避免浏览器 simple-request 触发本地 CSRF。
- 写入 Modelfile 前拒绝包含 `\\r/\\n` 的 `ggufPath`，避免 Modelfile 语义注入。
